Angular

Learning Angular was very frustrating because JS error messages and Angular JS error messages in particular are nearly useless in diagnosing the issue. I had become so used to the nice concise error messages that Rails provided that transitioning was very difficult. Also, many instructions online related to Angular 2, which is completely different from Angular 1.

Once I had Angular working, it was much easier to set up front-end components, especially those that had dynamically updating models and repeating lists that needed to be sorted or filtered. I would like to change some of the RoR applications I made earlier to use Angular, but converting all those html pages sounds like such a daunting task, especially since Angular 1 is on it’s way out. The next JS framework I’m going to tackle will be React, a framework similar to Angular, but on the rise. Flatiron offers React as part of it’s course now, but I chose to learn it on my own since there are so many other languages I need to teach myself and this one seems rather close to what I’m already familiar with.
